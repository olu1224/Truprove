III. Technical Architecture (The Engine Room)

The system is architected for security, low latency, and global scale, leveraging specialized encoding and blockchain immutability.

The TruProve Protocol: Technical Detail

Component Function Technical Specification
Hashing Engine Generates the unique digital fingerprint. Uses **SHA-256** standard. Calculation occurs client-side or in ephemeral memory.
Blockchain Anchoring Creates immutable, public timestamp. Smart Contract stores only the **SHA-256 hash** and User ID. Uses a cost-effective L2 chain (e.g., Polygon).
Digital Media Proof (DMP) The compact, user-facing verification code. Custom **Base44 encoding** compresses the raw Hash + TxID into a **28-character URL-safe string** (e.g., `https://truprove.co/verify/[Code]`).
TruProve Shield (Phase 2 R&D) Deepfake Detection API. Proprietary **ML model (CNNs)** for real-time video/image manipulation analysis, outputting a **Synthetic Confidence Score (SCS)**.
